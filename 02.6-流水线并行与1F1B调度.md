# 02.6 · 流水线并行与 1F1B 调度（Pipeline Parallel）

> 本篇是 [02 · 并行化子系统](./02-并行化子系统.md) 的**子文档**，承接 [02.4 · 并行组构建与通信详解](./02.4-并行组构建与通信详解.md)、[02.5 · 张量并行实现详解](./02.5-张量并行实现详解.md)。本篇聚焦：流水线并行（PP）如何把模型按层切到不同 stage、为什么需要 1F1B 调度、warmup/稳态/cooldown 三阶段的精确执行顺序，以及底层 P2P 通信。

> - `tp_group` 之于 TP，`pp_group` 之于 PP：均来自 [02.4](./02.4-并行组构建与通信详解.md) 的 `parallel_state`

## 1. 流水线并行在切什么

TP 切的是「单层内部的矩阵」（见 02.5），**PP 切的是「层的深度」**：把一个 N 层的模型按层段分给不同 GPU（称为 **stage**）。

```
完整模型: [Layer 0..7]   PP=4 切成 4 个 stage:
  stage 0 (rank a): Layer 0,1   ← 含 embedding
  stage 1 (rank b): Layer 2,3
  stage 2 (rank c): Layer 4,5
  stage 3 (rank d): Layer 6,7   ← 含 输出层 / loss
```

数据像在流水线上流动：stage 0 算完把激活 **P2P 发给** stage 1，stage 1 接着算……到最后一个 stage 算 loss；反向梯度沿相反方向逐级传回。**相邻 stage 间只传一个激活张量**，通信量远小于 TP，所以 PP 可以跨节点（见 02.4 中 `order` 把 pp 放最右、stride 最大）。

`num_warmup_microbatches` 取决于本 stage 的位置（`schedules.py:2230`）：
```python
num_warmup_microbatches = total_stages - current_stage - 1
```

---

## 2. 朴素流水线的问题：气泡（bubble）

若一次只送一个 batch：stage 0 工作时，stage 1/2/3 全在干等；只有当激活传到最后一级，前面又空了。GPU 利用率极低——这段空闲叫 **pipeline bubble（流水线气泡）**。

```
朴素 (1 个 batch, PP=4):
stage0: F . . . . . . B
stage1: . F . . . . B .
stage2: . . F . . B . .
stage3: . . . F B . . .
        ↑ 大量空白 = 气泡
```

---

## 3. 解法：microbatch + 1F1B 调度

把一个全局 batch 切成 M 个 **microbatch**，让它们像车流一样连续灌入流水线，把气泡摊薄。

- **GPipe（朴素）**：先做完所有 M 个 forward，再做所有 M 个 backward → 需要同时保存 M 份激活，**显存峰值高**。
- **1F1B（One-Forward-One-Backward，Megatron 采用）**：进入稳态后，**每做一个 forward 就紧跟一个 backward**，让激活尽快被反向消费掉释放 → 同一时刻只需保存约 `num_warmup` 份激活，**显存峰值大幅下降**。

1F1B 把每个 stage 的执行分成**三个阶段**：

```
某 stage 的一生：
  [warmup 暖机]   连续做 num_warmup 个 forward（只前向，攒够在途激活）
  [steady 稳态]   1F1B：forward 一个 → backward 一个，交替进行
  [cooldown 收尾] 把剩下 num_warmup 个 backward 做完，排空流水线
```

`num_warmup = total_stages - current_stage - 1`：**越靠前的 stage 暖机越多**（stage 0 暖机 3 个，最后一个 stage 暖机 0 个），这样最后一级一拿到激活就能立刻 1F1B。

---

## 4. 经典 1F1B 时间线（PP=4, M=8）

```
时间 →
stage0: F1 F2 F3 F4 B1 F5 B2 F6 B3 F7 B4 F8 B5    B6    B7    B8
stage1:    F1 F2 F3 B1 F4 B2 F5 B3 F6 B4 F7 B5 F8 B6    B7    B8
stage2:       F1 F2 B1 F3 B2 F4 B3 F5 B4 F6 B5 F7 B6 F8 B7    B8
stage3:          F1 B1 F2 B2 F3 B3 F4 B4 F5 B5 F6 B6 F7 B7 F8 B8
        └warmup┘ └──────────── steady 1F1B ────────────┘ └cooldown┘
        Fn=第n个microbatch前向  Bn=第n个microbatch反向
```

注意 stage3（最后一级）warmup=0，立刻 F1→B1 交替；stage0 warmup=3，先攒 3 个前向。**稳态区每个 stage 都满负荷**，气泡被压缩到首尾两段。

---

## 5. 代码逐段对照（`forward_backward_pipelining_without_interleaving`）

调度由 `get_forward_backward_func()`（`schedules.py:48`）按 PP/VPP 配置选出（见 [02.4 §3 与 01 §8.4](./01-框架透视图解.md)）。非交错经典版主体在 `schedules.py:2092`，三阶段一一对应：

### 5.1 Warmup（`schedules.py:2287`）
```python
for i in range(num_warmup_microbatches):
    input_tensor = p2p_communicator.recv_forward(...)      # 从上一级收激活
    output_tensor, _ = forward_step(...)                   # 本级前向
    p2p_communicator.send_forward(output_tensor, ...)      # 发给下一级
    input_tensors.append(input_tensor)                     # 暂存，留给将来反向
    output_tensors.append(output_tensor)
```

### 5.2 Steady 1F1B（`schedules.py:2332`）—— 核心
```python
input_tensor = p2p_communicator.recv_forward(...)          # 先收稳态第一个激活
for i in range(num_microbatches_remaining):
    output_tensor, _ = forward_step(...)                   # ① 一次 forward

    # ★ 关键：一次通信同时"发前向激活 + 收反向梯度"
    output_tensor_grad = p2p_communicator.send_forward_recv_backward(output_tensor, ...)

    input_tensors.append(input_tensor)
    output_tensors.append(output_tensor)
    input_tensor  = input_tensors.pop(0)                   # 取最早的激活
    output_tensor = output_tensors.pop(0)

    input_tensor_grad = backward_step(                     # ② 紧跟一次 backward
        input_tensor, output_tensor, output_tensor_grad, config)

    if last_iteration:
        p2p_communicator.send_backward(input_tensor_grad, ...)
    else:
        # ★ 再次融合："发反向梯度 + 收下一个前向激活"
        input_tensor = p2p_communicator.send_backward_recv_forward(input_tensor_grad, ...)
```

> `send_forward_recv_backward` / `send_backward_recv_forward` 是 1F1B 的灵魂：把「发」和「收」打包进**同一次** `batch_isend_irecv`，既避免死锁，又让通信与下一步计算重叠。

### 5.3 Cooldown（`schedules.py:2405`）
```python
for i in range(num_warmup_microbatches):
    input_tensor  = input_tensors.pop(0)
    output_tensor = output_tensors.pop(0)
    output_tensor_grad = p2p_communicator.recv_backward(...)   # 收反向梯度
    input_tensor_grad  = backward_step(...)                    # 把暖机期攒的前向逐个反向
    p2p_communicator.send_backward(input_tensor_grad, ...)     # 发回上一级
```

### 5.4 收尾：跨 stage 的梯度同步（`schedules.py:2435`）
所有 backward 做完后调用 `config.finalize_model_grads_func`，完成：DP 的梯度 all-reduce/reduce-scatter、序列并行的 layernorm all-reduce、以及 **PP 首尾 stage 共享词嵌入权重的 embedding all-reduce**（`_EMBEDDING_GROUP` 即为此建，见 02.4 §1.4）。

## 6. 底层 P2P 通信（`p2p_communication.py`）

PP 不用集合通信，而用**点对点**收发，全部基于异步原语：
```python
# p2p_communication.py:29 起
send_op = torch.distributed.P2POp(torch.distributed.isend, tensor, next_rank, group=pp_group)
recv_op = torch.distributed.P2POp(torch.distributed.irecv, buffer, prev_rank, group=pp_group)
reqs = torch.distributed.batch_isend_irecv([send_op, recv_op])   # 打包一起发起
```
- `group` 就是 `parallel_state.get_pipeline_model_parallel_group()` 提供的 `pp_group`（如 02.4 例子里的 `[0,4]`）。
- 发往 `next_rank`、收自 `prev_rank`——即流水线相邻级。
- 融合算子（`send_forward_recv_backward` 等，`p2p_communication.py:527` 起）把多个 `P2POp` 放进一个 `batch_isend_irecv`，由 NCCL 一次性调度，避免「你等我发、我等你发」的死锁。
- 首次通信前还会用 `_communicate_shapes`（`:186`）交换张量形状。

---

## 7. 气泡占比与权衡

理想气泡占比（bubble fraction）≈ `(PP - 1) / M`：
- M（microbatch 数）越大，气泡越小，但激活显存与调度开销上升。
- PP 越大，气泡越大；通常用 **更大的 M** 来摊薄。

1F1B 相比 GPipe **不改变气泡占比**，但把**激活显存峰值**从 `O(M)` 降到 `O(num_warmup) ≈ O(PP)`——这是它的核心价值。

---

## 8. 延伸：交错式 1F1B（Interleaved / 虚拟流水线）

`forward_backward_pipelining_with_interleaving`（`schedules.py:949`）让每个物理 stage 持有**多个不连续的层段（virtual pipeline / model chunk）**。例如 stage 0 同时拿 Layer 0,1 和 Layer 8,9。代价是 P2P 通信次数变多，收益是**进一步缩小气泡**（暖机式 `num_warmup` 见 `schedules.py:894`，含 `num_model_chunks` 因子）。由 `--num-layers-per-virtual-pipeline-stage` 触发，对应配置里的 `virtual_pipeline_model_parallel_size`（02.4 §1.4 中 `_VIRTUAL_PIPELINE_*` 全局状态）。

---

## 9. 小结

- PP 按**层深度**切模型成 stage，相邻 stage 用 **P2P** 传激活/梯度，通信轻、可跨节点。
- 朴素流水线有大量**气泡**；解法是切 **microbatch** + **1F1B 调度**。
- 1F1B 三阶段：**warmup（连续前向）→ steady（前向/反向交替）→ cooldown（排空反向）**，`num_warmup = total_stages - current_stage - 1`。
- 1F1B 的灵魂是 `send_forward_recv_backward` / `send_backward_recv_forward`——一次 `batch_isend_irecv` 同时收发，避免死锁、重叠通信。
- 1F1B 不改变气泡占比，但把激活显存峰值从 `O(M)` 降到 `O(PP)`；交错式可进一步缩小气泡。
- 所有 P2P 落在 `parallel_state` 提供的 `pp_group` 上；首尾 stage 的词嵌入通过 `_EMBEDDING_GROUP` all-reduce 同步。
