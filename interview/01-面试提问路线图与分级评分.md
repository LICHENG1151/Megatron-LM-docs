## 1. 分级标尺（L0–L3）

用**同一个知识点**在 4 个深度上区分候选人。判断依据不是"答没答对"，而是"能到哪一层"。

| 级别 | 含义 | 典型信号（以「TP 列切/行切」为例） |
|------|------|-----------------------------------|
| **L0 认知** | 知道术语、会用配置 | "TP 就是把权重切到多卡，设 `--tensor-model-parallel-size`" |
| **L1 机制** | 知道怎么实现的 | "列切按输出维、行切按输入维；MLP 是列切→行切，中间 all-reduce" |
| **L2 原理** | 能解释为什么、会算通信/显存账 | "行切要 all-reduce 是因为切在收缩维产生部分和；一层前向 2 次、反向 2 次，是最小值" |
| **L3 设计** | 能推导、选型、排障、迁移 | "f/g 互为伴随所以梯度自动正确；把 GeLU 换成跨特征归约就不再免通信；TP=8 通信占比升高时应转 PP" |

> **关键分水岭在 L1→L2**：能复述实现（L1）的人很多，能说清"为什么非这样不可、换个条件会怎样"（L2/L3）的人少。面试重心放在把候选人从 L1 往 L2/L3 推。

## 2. 能力轴（5 条）

不要只沿 TP 一条线深挖——那样只能筛出"背了 TP 的人"。先横向扫 5 条轴定位强弱，再纵向深钻。每条轴拆成若干**子能力**，方便定位候选人到底强在哪一块。

|----|--------------|--------------|
| **A. 并行策略** | ① 列切/行切的切法与通信 ② f/g 对偶与反向 ③ MLP/Attention 组合 ④ SP 与通信守恒 ⑤ PP 调度与气泡 ⑥ EP/CP 的适用与通信 ⑦ 多维并行组合与放置 | Q1–Q9, Q14–Q16, Q24–Q26 ｜ [02.5](../02.5-张量并行实现详解.md)[02.6](../02.6-流水线并行与1F1B调度.md)[02.7](../02.7-上下文并行.md)[02.8](../02.8-专家并行.md) |
| **B. 显存与激活** | ① 16B/param 显存账 ② 激活为何是大头 + 重计算 ③ fp32 master weight ④ 分布式优化器/ZeRO 切分 ⑤ 梯度累积 | [04·Q1–Q6](./04-分布式训练与优化器面试题集.md) ｜ [02.1](../02.1-显存、激活值与重计算.md)[02.3.1](../02.3.1-FSDP.md) |
| **C. 通信** | ① 5 原语语义 ② `AllReduce=RS+AG` 与通信量 ③ 计算/通信重叠与分桶 ④ 带宽层级与并行放置 | [02·Q12–Q13](./02-并行化子系统面试题集.md), [04·Q7–Q8](./04-分布式训练与优化器面试题集.md) ｜ [02.3](../02.3-通信原语速查.md) |
| **D. 训练流程** | ① 一次 step 的完整数据流 ② 词表并行 + 并行交叉熵 ③ 全局梯度裁剪 ④ loss scaling ⑤ 分布式检查点/重切分 ⑥ 可复现续训 | [02·Q10–Q11](./02-并行化子系统面试题集.md), [06·Q1](./06-训练框架Harness面试题集.md), [04·Q9–Q10](./04-分布式训练与优化器面试题集.md), [08·Q1–Q2](./08-检查点与重切分面试题集.md) ｜ [02.0](../02.0-Transformer与MoE结构基础.md)[08](../08-检查点与重切分.md) |
| **E. 工程与排障** | ① 吞吐/MFU 定位 ② OOM 手段与代价排序 ③ TP 过大的退化 ④ NaN 排查 ⑤ 关键环境变量与坑 | [02·Q9](./02-并行化子系统面试题集.md), [06·Q2–Q5](./06-训练框架Harness面试题集.md) ｜ [06](../06-训练框架Harness.md) |

> **用法**：Phase 1 广度扫描时，每条轴挑 1 个子能力问；Phase 2 深钻时，顺着某条轴的子能力①→⑦逐个加深。子能力也可直接作为[§6 评分表](#6-评分表模板)里"关键证据"的记录锚点。

## 2.1 能力矩阵：能力轴 × 能力等级 ★

> **本表是这套面试的核心工具（第一版，后续按面试反馈迭代）。** 纵轴 = 5 条能力轴，横轴 = 4 个等级；每格写清**该级在该轴应具备的能力**与**判定它的关键特征（"tell"）**。面试时对照落格：候选人在某轴能稳定做到哪一格，就是该轴的水平。
>
> 读法：每格上行=**能做到什么**，下行=*关键特征/判据*。分水岭见 [§1](#1-分级标尺l0l3)：**L1→L2（会复述 → 会推导）** 是最重要的一跳。

### A. 并行策略

| L0 认知 | L1 机制 | L2 原理 | L3 设计 |
|---------|---------|---------|---------|
| 知道 TP/PP/DP/EP/CP 术语，会设并行度参数 | 说清列切/行切切哪维、MLP=列→行、通信在何处；PP 切层、DP 切数据 | 推导行切为何 all-reduce、中间为何免通信、一层 2+2 次是最小；会算 SP 通信守恒 | 设计多维并行组合与放置；迁移到新条件（GQA/反序/EP vs TP）；从通信模式反推结构 |
| *只会概念+配置项* | *能复述实现步骤* | *能答"为什么"、会算通信账* | *能选型+迁移+反推* |

### B. 显存与激活

| L0 认知 | L1 机制 | L2 原理 | L3 设计 |
|---------|---------|---------|---------|
| 知道"大模型装不下要切"，听过重计算/ZeRO | 拆 16B/param 账；知道激活是大头、重计算=算力换显存、ZeRO 切优化器状态 | 解释 fp32 master 为何必要、分布式优化器为何通信守恒、梯度累积为何不增显存 | 按场景权衡 ZeRO-1/2/3 vs TP；估算某配置的显存占用；组合选型 |
| *只知道有这些手段* | *能拆账、说清各手段做什么* | *能讲机制背后的数值/通信原因* | *能定量估算+权衡组合* |

### C. 通信

| L0 认知 | L1 机制 | L2 原理 | L3 设计 |
|---------|---------|---------|---------|
| 知道多卡要通信，听过 all-reduce | 说清 5 原语"做完后每卡有什么"；知道各并行用哪种原语 | 理解 `AllReduce=RS+AG`、≈2S 与卡数无关；懂重叠/分桶把通信藏进计算 | 按带宽层级（NVLink/IB）做并行放置决策；判断通信是否瓶颈并给优化 |
| *只有模糊概念* | *能把原语↔策略对号入座* | *会算通信量、懂重叠原理* | *能把通信特征落到部署决策* |

### D. 训练流程

| L0 认知 | L1 机制 | L2 原理 | L3 设计 |
|---------|---------|---------|---------|
| 知道训练是"前向-反向-更新"，会跑训练脚本 | 描述完整 step 数据流；知道混合精度/梯度累积/检查点的存在 | 解释全局梯度裁剪为何跨组、loss scaling 为何必要、检查点为何要 reshard | 设计可复现续训（RNG/数据位置）；跨并行度迁移 ckpt；诊断流程类 bug |
| *黑盒使用* | *能串起流程各环节* | *懂每环节的分布式/数值细节* | *能覆盖易漏项+排障* |

### E. 工程与排障

| L0 认知 | L1 机制 | L2 原理 | L3 设计 |
|---------|---------|---------|---------|
| 遇问题靠重启/加卡/调 batch 试 | 知道用 profiler、知道 OOM 有哪些手段、知道 `CUDA_DEVICE_MAX_CONNECTIONS` 这类变量 | 按占比对症（先 profiler 再定位重叠/bubble/TP）；按代价排序 OOM 手段 | 定位复杂退化（TP 过大跨 NVLink、多机梯度不同步致 NaN）；系统性二分复现+优化闭环 |
| *无系统方法* | *知道有工具/手段但不成体系* | *有排查路径和优先级* | *能根因定位+闭环* |

> **怎么用这张表**：
> - **横向读**看深度：候选人在一条轴上能推进到哪一格。
> - **纵向读**看广度：同一等级下，5 条轴是否均衡，还是偏科（如 A 到 L3 但 B/E 停 L0，典型"只钻过 TP 源码、没跑过真实训练"）。
> - 结合 [§5 变条件追问](#5-防背题变条件迁移追问)：能进 L2/L3 格**且扛住变条件**才算数，否则降一格记。
> - **迭代**：面试后把"实际出现但本表没覆盖的强/弱信号"补进对应格的*关键特征*行，逐步校准。

总时长约 **45–60 min**，四段。核心是"**广度定位 → 在强项上深钻到答不出 → 综合验证**"。

```
Phase 0  破冰定起点        (5 min)  ── 让候选人自述，决定从哪条轴、哪一级切入
Phase 1  广度扫描          (10 min) ── 每条轴 1 个 L0/L1 锚点，快速画出能力雷达
Phase 2  深度钻取          (20 min) ── 挑 2 条最强轴，用追问阶梯逐级加深到 L3 或卡住
Phase 3  综合设计/排障      (15 min) ── 开放场景题，验证 L3 是真能力还是背诵
```

### Phase 0 · 破冰定起点（5 min）
- "你在 Megatron 里实际训过多大的模型？用过哪些并行维度？哪一块你改过/调过？"
- 作用：**定起点**。声称改过 TP 的直接从 Phase 2 的 A 轴阶梯 R2 起步；只用过默认配置的从 Phase 1 L0 起步。

### Phase 1 · 广度扫描（10 min，每轴一问）
| 轴 | 锚点问题（L0–L1） | 想听到 |
|----|------------------|--------|
| A | 列切、行切分别切权重哪一维？MLP 怎么组合？（Q1） | 输出维/输入维；列切→行切 |
| B | fp16 + Adam，1 个参数训练态占多少字节？（16B 账） | 参数2+梯度2+优化器状态12 |
| C | all-reduce / all-gather / reduce-scatter，做完每卡手上是什么？（Q12） | 三者语义清晰 |
| D | 词表并行下，输出层 logits 怎么算 loss 不用聚合全量？（Q11） | 并行交叉熵思路 |
| E | 训练变慢/OOM 时你先看什么？ | profiling、通信占比、激活显存 |

> 每问只花 1–2 min，**不深究**，目的是画出雷达图，找出 2 条最强轴带进 Phase 2。

### Phase 2 · 深度钻取（20 min）
在候选人最强的 2 条轴上，走[§4 追问阶梯](#4-追问阶梯核心)，**一级一级往下问，直到答不出为止**——他停在哪一级，就是那条轴的真实水平。中途穿插[§5 变条件追问](#5-防背题变条件迁移追问)验证是不是真懂。

### Phase 3 · 综合设计/排障（15 min）
选 1–2 个开放场景（见 [§4 阶梯 E](#阶梯-e综合设计--排障l2l3)），看他能否**跨轴串联、讲权衡、给排查路径**。这是 L3 的试金石。

## 4. 追问阶梯（核心）

每条阶梯 = 一个主题的 4 级追问链，从现象问到边界。**R1→R4 逐级深入**，每级标了它对应的分级与题号，以及"上一级答出后该怎么再往下钻"。

### 阶梯 A：TP 列切/行切（并行策略主线）
| 级 | 追问 | 对应 |
|----|------|------|
| R1 (L0) | 列切、行切各切权重哪一维？ | Q1 |
| R2 (L1) | 一个 MLP 为什么是「列切→行切」？前向几次通信？ | Q5, Q7 |
| R3 (L2) | 为什么中间 GeLU 不用通信？行切前向为什么必须 all-reduce？ | Q2, Q5 |
| R4 (L3) | 反向的通信在哪一层？为什么说 f/g 互为伴随、梯度会自动正确？ | Q3, Q25 |
| ➜ 变条件 | 见 §5-①②（GeLU→LayerNorm；顺序反过来） | Q24 |

### 阶梯 E：综合设计 / 排障（L2–L3）
开放题，无标准答案，看**权衡意识**与**排查路径**：
- **选型**："512 卡训一个 70B dense 模型，你怎么配 TP/PP/DP(+SP)？说清每一维为什么这么定、边界在哪。"（好答案：TP≤单节点GPU数走NVLink、PP跨节点、DP补满、SP省激活；能点出 TP 通信在关键路径所以不宜过大）→ [04·Q8](./04-分布式训练与优化器面试题集.md), [06·Q4](./06-训练框架Harness面试题集.md)
- **吞吐排障**："loss 正常但 MFU 很低，你按什么顺序定位？"（想听到：先 profiler 看通信/计算占比 → 检查通信是否重叠、bubble、DP 通信是否被 overlap、数据加载是否瓶颈）→ [06·Q2](./06-训练框架Harness面试题集.md)
- **OOM 排障**："训练 OOM，你有哪些手段？按代价从小到大排。"（重计算 → SP → 减 microbatch/增梯度累积 → 提 TP/PP → ZeRO/offload；能讲清每招换来的通信或速度代价）→ [06·Q3](./06-训练框架Harness面试题集.md)
- **NaN 排障**："多机 loss 突然 NaN，怎么系统性二分定位？"（loss scale / 坏数据 / lr / 梯度不同步 / 硬件）→ [06·Q5](./06-训练框架Harness面试题集.md)
- **MoE 扩展**："加 MoE 后通信变化在哪？EP 和 TP 有什么本质不同？"（Q15：all-to-all vs all-reduce、负载不均、数据依赖）

## 5. 防背题：变条件迁移追问

**这是区分"背题"与"真懂"的关键。** 每个锚点都准备一个改前提的追问——背过标准答案的人会卡住，理解本质的人能当场推。

| # | 原问题 | 变条件追问 | 真懂的人会说 |
|---|--------|-----------|-------------|
| ① | MLP 中间 GeLU 免通信（Q5） | "如果激活函数换成需要跨 hidden 维归约的操作（如带全局统计的 LayerNorm），还免通信吗？" | 不免了——归约要跨分片，必须先聚合；这正说明"能免通信"依赖逐元素性质 |
| ② | 列切→行切（Q5） | "反过来行切→列切会怎样？"（Q24） | 中间非线性 `f(Σ)≠Σf`，被迫多一次 all-reduce，激活也变回完整态 |
| ③ | QKV 按头列切（Q6） | "如果是 MQA/GQA（KV 头远少于 Q 头），QKV 还能简单按头切吗？" | KV 头数可能 < TP size，需复制/特殊切分，考察对 head 维切分边界的理解 |
| ④ | 词表并行置零求和（Q10） | "如果词表大小不能被 world_size 整除怎么办？" | 需 padding 到可整除，或末卡少分；考察是否只会背"独热求和"话术 |
| ⑤ | f/g 前向/反向（Q3） | "只给你'前向 all-reduce、反向不通信'，这是列切还是行切？"（反推） | 行切出口 g；能从通信模式反推结构=真懂 |
| ⑥ | 一层 2+2 次 all-reduce（Q7） | "如果开了 SP，这个计数和通信量怎么变？" | 次数换成 RS+AG、总量不变、激活分片；串联 A/C 两轴 |
| ⑦ | 手写（Q21/Q22） | 白板写 f/g 的 `autograd.Function`，或 TP-MLP 前向 | 能写出 `backward` 正确返回 all-reduce/None，且 bias 在 reduce 后加 |

> 用法：候选人在某级答得很顺时，**立刻抛对应变条件**。顺→真懂，往下钻；卡→他停在背诵层，记为该级"未通过"。

## 6. 评分表模板

每条轴记录**到达级别**与**是否扛住变条件**，避免"感觉不错"的模糊判断。

| 能力轴 | 到达级别 (L0–L3) | 扛住变条件? | 关键证据（一句话） |
|--------|:---:|:---:|--------------------|
| A 并行策略 | | | |
| B 显存与激活 | | | |
| C 通信 | | | |
| D 训练流程 | | | |
| E 工程与排障 | | | |

**判级速记**：
- 只会复述配置/术语 → **L0**
- 能讲实现步骤但问"为什么"就停 → **L1**
- 能推导原因、会算通信/显存账 → **L2**
- 能迁移到新条件、能选型/排障 → **L3**

**红旗**（背题信号）：一改前提就答不上；能背通信次数却说不清为什么；只会顺着讲、无法反推。
**加分**（真懂信号）：主动画通信/显存账；未问就提到权衡与边界；能从通信模式反推结构。

## 7. 按岗位定 Hire Bar

同一套题，不同岗位看不同轴、不同级别达标线。

| 岗位画像 | 必达 | 加分 | 说明 |
|----------|------|------|------|
| **框架使用者**（跑训练/调参） | A/B/D 到 **L1** | 任一轴 L2 | 会配、会用、能看懂日志与 OOM 即可 |
| **框架开发/Infra** | A/C 到 **L2**，且扛住 ≥2 个变条件 | A 或 C 到 L3、能手写 f/g | 要能改实现、加特性、定位通信 bug |
| **性能优化** | C/E 到 **L2–L3** | 能给完整吞吐/显存排障路径 | 重点在通信重叠、bubble、MFU、选型权衡 |

> 结论建议：**任一"必达轴"停在 L0，或所有变条件全部卡住**，视为该岗位不达标——多半是"读过文档/背过题"而非真正掌握。
