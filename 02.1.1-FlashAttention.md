# 02.1.1 · FlashAttention（把注意力激活从 O(s²) 压到 O(s)）

> 本篇是 [02.1 · 显存、激活值与重计算](./02.1-显存、激活值与重计算.md) 的**技术专题**（同属〔铺垫〕层）。02.1 §5 讲了"重计算 = 用算力换激活显存"，并说 FlashAttention 是它"在注意力上的名场面"。这里把这句话**完整展开**：从背景、问题、数学原理讲到 Megatron 里的具体实现。
>
> 建议在 [02.1](./02.1-显存、激活值与重计算.md)（重计算）和 [02.0 §2](./02.0-Transformer与MoE结构基础.md)（注意力结构）之后读；本篇也是 [02.7 §3 Ring Attention](./02.7-上下文并行与专家并行.md) 的数学前置。
>
> 面向初学者：只需要"softmax、矩阵乘、以及 02.1 的重计算直觉"。

## 0. 记号约定

| 符号 | 含义 |
|------|------|
| `s` / `d` | 序列长度 / 每个注意力头的维度（head dim） |
| `Q, K, V` | 查询/键/值矩阵，均为 `[s, d]`（单头视角） |
| `S = QKᵀ/√d` | 注意力打分矩阵，`[s, s]` |
| `P = softmax(S)` | 注意力权重矩阵（逐行 softmax），`[s, s]` |
| `O = P·V` | 注意力输出，`[s, d]` |
| `Br / Bc` | 行块/列块大小；`Tr = s/Br`、`Tc = s/Bc` 为块数 |
| **HBM / SRAM** | GPU 高带宽显存（大、慢）/ 片上共享内存（小、极快） |

> 全篇以**单头、单 batch** 视角写；多头/batch 只是把同样的计算并行复制。

---

## 1. 背景：标准注意力为什么又慢又占显存

一个注意力头的标准三步（[02.0 §2](./02.0-Transformer与MoE结构基础.md)）：

```
① S = QKᵀ/√d      # [s,d]·[d,s] → [s,s]   打分
② P = softmax(S)   # 逐行 softmax           → [s,s]   归一
③ O = P·V          # [s,s]·[s,d] → [s,d]   加权求和
```

问题出在中间那两个 **`[s, s]` 的大矩阵 `S` 和 `P`**：

- **显存**：`S`、`P` 都是 `O(s²)`。序列 `s=8K` 时，单头单条序列的 `P` 就是 `8192² ≈ 6700 万` 个元素；乘上 batch、头数、层数，激活显存直接爆（回指 [02.1 §2.2](./02.1-显存、激活值与重计算.md)：激活 ∝ `b·s·h·L`，注意力里还额外 ∝ `s²`）。
- **速度（关键但反直觉）**：现代 GPU 的算力（FLOPs）远远过剩，真正的瓶颈是**把数据在 HBM 和 SRAM 之间搬来搬去（IO）**。标准实现把 `S`、`P` 这两个 `s×s` 巨物**整个写回 HBM 再读出来**（softmax 读一遍 `S`、写一遍 `P`，`PV` 又读一遍 `P`），HBM 读写量是 `O(s²)`。注意力是典型的 **memory-bound（访存受限）** 算子——时间几乎全花在搬 `s²` 数据上，而非算数。

> 一句话背景：`s×s` 中间矩阵既撑爆显存，又制造大量 HBM 读写，是长序列注意力的两座大山。

---

## 2. 问题：能不能"不物化整个 s×s 矩阵"

标准实现的根子在于**先算完整的 `S`、`P`，再算 `O`**——必须把 `s×s` 落到 HBM。两个诉求：

1. **省显存**：前向别保存 `O(s²)` 的 `S`/`P`（否则反向还要用，长期占显存）。
2. **省 IO**：别把 `s×s` 反复在 HBM↔SRAM 之间搬。

FlashAttention 的答案：**分块（tiling）+ online-softmax + 反向重算**——把 `Q/K/V` 切成能塞进 SRAM 的小块，在片上一次算完一小块的 `S→softmax→PV` 并**增量累加**到输出，全程不把完整 `S`/`P` 写回 HBM。难点是：**softmax 需要一整行的全局信息（分母要对整行求和），怎么能分块算？** 这正是数学原理要解决的。

---

## 3. 数学原理

### 3.1 online-softmax：softmax 可以分块增量计算

先看一行 softmax（数值稳定版，减去最大值防溢出）。设一行打分为 $x_1,\dots,x_s$：

$$
m=\max_j x_j,\qquad \ell=\sum_{j} e^{x_j-m},\qquad \text{softmax}(x)_j=\frac{e^{x_j-m}}{\ell}.
$$

把这一行**切成若干块**依次处理。维护两个"运行统计量"：当前见过的最大值 $m$ 与当前的指数和 $\ell$。来了新的一块 $x^{(\text{new})}$，其块内最大值为 $\tilde m$、块内指数和为 $\tilde\ell=\sum e^{x^{(\text{new})}-\tilde m}$，则更新公式为：

$$
\boxed{\;m' = \max(m,\tilde m),\qquad
\ell' = e^{\,m-m'}\,\ell \;+\; e^{\,\tilde m-m'}\,\tilde\ell\;}
$$

**关键：$e^{m-m'}$ 这个"缩放因子（rescale）"把旧的 $\ell$ 修正到新基准 $m'$ 上。** 因为 $e^{x-m}=e^{x-m'}\cdot e^{m'-m}$，换基准只需乘一个标量。逐块套用，处理完整行后的 $(m,\ell)$ 与一次性算的完全相等——**softmax 的分母可以在只看过部分数据时就正确地增量累加**。

### 3.2 把输出 O 也增量累加（不物化 P）

注意力输出某一行是 $O=\sum_j P_j V_j=\frac{1}{\ell}\sum_j e^{x_j-m}V_j$。定义**未归一化**的累加量 $\tilde O=\sum_j e^{x_j-m}V_j$，它也能像 $\ell$ 一样分块 rescale 累加。处理新块 $(x^{(\text{new})}, V^{(\text{new})})$ 时：

$$
\tilde O' = e^{\,m-m'}\,\tilde O \;+\; \sum_{j\in\text{new}} e^{\,x_j-m'}\,V_j,
\qquad
O = \frac{\tilde O}{\ell}\ \text{（全部块处理完再除一次）}.
$$

于是**外层遍历 Q 的行块、内层遍历 K/V 的列块**，每次在 SRAM 里算一小块 $S_{ij}=Q_iK_j^\top/\sqrt d$，更新 $(m_i,\ell_i,\tilde O_i)$——**从头到尾没有完整的 `S` 或 `P` 出现在 HBM**，激活显存里只留 `O(s·d)` 的 `O` 和每行一个标量统计量。伪代码：

```
for i in range(Tr):                      # 遍历 Q 行块 Q_i:[Br,d]
    m_i, ℓ_i = -inf, 0;  Õ_i = 0         # 该行块的运行统计量
    for j in range(Tc):                  # 遍历 K/V 列块 K_j,V_j:[Bc,d]
        S_ij = Q_i @ K_j.T / sqrt(d)     # [Br,Bc]，只在 SRAM
        m̃ = rowmax(S_ij);  P̃ = exp(S_ij - max(m_i, m̃))
        m_new = max(m_i, m̃)
        ℓ_i = exp(m_i - m_new)*ℓ_i + rowsum(P̃)       # rescale + 累加分母
        Õ_i = exp(m_i - m_new)*Õ_i + P̃ @ V_j          # rescale + 累加输出
        m_i = m_new
    O_i = Õ_i / ℓ_i                      # 收尾归一
    L_i = m_i + log(ℓ_i)                 # 存 logsumexp 供反向（见 3.3）
```

### 3.3 反向：不存 P，靠重算（这就是 02.1 §5 的重计算）

反向要 `dQ/dK/dV`，公式里都要用到前向的 `P`。FlashAttention **不保存 `P`**（那是 `O(s²)`），只保存：输出 `O`（`O(s·d)`，本来就要留）和每行的 **logsumexp 统计量 `L = m + log ℓ`**（`O(s)`）。反向时用 `Q,K,V,L` **分块重算** `P_ij = exp(S_ij − L_i)`，再算梯度：

$$
dV_j = \sum_i P_{ij}^\top\, dO_i,\quad
dP_{ij}=dO_i V_j^\top,\quad
dS_{ij}=P_{ij}\odot\big(dP_{ij}-D_i\big),\quad D_i=\text{rowsum}(dO_i\odot O_i),
$$

再由 $dS$ 得 $dQ,dK$。这就是 [02.1 §5](./02.1-显存、激活值与重计算.md) 说的**"用一次额外计算换激活显存"**：注意力激活从 `O(s²)` 降到 **`O(s)`**（只留 `O` 与 `L`）。

> 小结数学：**online-softmax 让 softmax 可分块**（3.1）→ **输出可 rescale 增量累加**（3.2）→ **反向靠 `L` 重算 P**（3.3）。三者合起来 = 全程不物化 `s×s`，显存 `O(s²)→O(s)`，HBM 读写也从 `O(s²)` 降到 `O(s²/M)`（`M`=SRAM 容量，块越大搬得越少）。

## 4. 具体实现：Megatron 里怎么开 FlashAttention

### 4.1 训练路径：委托给 Transformer Engine（TE），不自己实现

Megatron **训练前向不自己写 FlashAttention 内核**，而是把 `core_attention` 交给 **Transformer Engine**：

- `TEDotProductAttention`（`megatron/core/extensions/transformer_engine.py:1583`）继承 `te.pytorch.DotProductAttention`；TE 内部再派发到它自己的 FlashAttention / cuDNN fused / unfused 内核。Megatron 只负责**选后端**。
- 朴素回退实现 `DotProductAttention`（`megatron/core/transformer/dot_product_attention.py`）是**纯 PyTorch 的 softmax-matmul**（会物化 `s×s`），**不是** FlashAttention，且不支持上下文并行（CP）。

### 4.2 用 `--attention-backend` 选后端

真正的开关是 **`--attention-backend`**（`TransformerConfig.attention_backend`，`transformer_config.py:143`，**默认 `auto`**），取值 `flash / fused / unfused / local / auto`。它经 `_set_attention_backend()`（`language_module.py:108`）翻译成 TE 读取的环境变量：

| `--attention-backend` | 行为 | 设置的环境变量 |
|----|----|----|
| `auto`（默认） | 让 TE 自己挑最优（flash/fused/unfused 都允许） | `NVTE_FLASH_ATTN=1, NVTE_FUSED_ATTN=1, NVTE_UNFUSED_ATTN=1` |
| `flash` | 强制 FlashAttention | `NVTE_FLASH_ATTN=1`，另两个=0 |
| `fused` | 强制 cuDNN fused attention | `NVTE_FUSED_ATTN=1`，另两个=0 |
| `unfused` | 强制未融合（会物化 `s×s`） | `NVTE_UNFUSED_ATTN=1`，另两个=0 |
| `local` | 走 mcore 朴素 PyTorch 实现 | 全 0；需配 `--spec local` |

- **`--use-flash-attn`** 是**遗留开关**（`arguments.py:2581`），实际后端选择已由 `--attention-backend` 接管；确定性模式下还禁止它（`arguments.py:1524`）。
- 注意：`local` 后端不支持 CP（长序列/超长上下文必须用 TE 的 flash/fused，见 [02.7](./02.7-上下文并行与专家并行.md)）。

### 4.3 推理/解码路径：直接调 flash_attn 系列内核

推理（尤其是 flash-decode）**直接调用外部 `flash_attn` 库**，不经 TE：`attention.py` 导入 `flash_attn`（FA2）、`flash_attn_3`/`flashattn_hopper`（FA3）、`flash_attn.cute`（FA4）、`flash_mla`（MLA 专用），在 `flash_decode()`（`attention.py:721`）里用 `flash_attn_with_kvcache(...)` 完成解码期的增量注意力。

> 对上号：本篇 §3 的"分块 + online-softmax + 反向重算"就是 TE / `flash_attn` 内核内部干的事；Megatron 的角色是**选后端 + 喂配置**。想理解它和长序列并行的接力，见 [02.7 §3](./02.7-上下文并行与专家并行.md)（Ring Attention 复用 online-softmax 沿序列跨卡累加）与 [02.8 §2](./02.8-进阶专题.md)（长序列工具箱）。

## 5. 小结

- **背景**：标准注意力的 `S`、`P` 是 `[s,s]` 巨物 → 激活 `O(s²)` + `O(s²)` 的 HBM 读写；注意力是访存受限（memory-bound）。
- **问题**：能否不物化完整 `s×s`？难点是 softmax 需要整行全局信息。
- **数学**：**online-softmax** 用运行 `max/sum` + rescale 因子 `e^{m−m'}` 让 softmax 可分块增量；输出 `Õ` 同样 rescale 累加；反向只存 `O` 与 `logsumexp L`、**分块重算 P**。合起来激活 `O(s²)→O(s)`，HBM 读写 `O(s²)→O(s²/M)`。
- **实现**：Megatron 训练**委托 TE**（`TEDotProductAttention`），用 **`--attention-backend`**（默认 `auto`，可选 `flash`）→ `NVTE_*` 环境变量选后端；`--use-flash-attn` 为遗留；`local` 回退是朴素实现、不支持 CP；推理路径直接调 `flash_attn`/`flash_attn_3`/`flash_mla`。
- **一句话**：FlashAttention = 02.1 重计算思想在注意力上的极致工程化——**分块算、不落 `s×s`、反向重算**。
